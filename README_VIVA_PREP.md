## 🎯 Answer Framework

For every question, use this structure:

**1. DEFINE** (10 seconds)
- What is X?
- Brief definition

**2. EXPLAIN** (20 seconds)
- How does it work?
- Why is it important?

**3. EXAMPLE** (20 seconds)
- Example from your project
- Show code if relevant

**4. ALTERNATIVES** (10 seconds - optional)
- Other approaches
- Why you chose this

**Total**: ~1 minute per question

## 📞 What If You Don't Know?

**If asked something you don't know:**

✅ **DO:**
- Say "I'm not completely sure about that"
- Explain what you DO know about related concepts
- Say "I would research X to find out"
- Show willingness to learn

❌ **DON'T:**
- Make up answers
- Get defensive
- Panic
- Say "I don't know" and stop talking

**Example:**
"I'm not completely familiar with Kubernetes deployment specifics, but I understand the concept of containerization with Docker. If I were to deploy to Kubernetes, I would research container orchestration patterns and follow the official documentation to create deployment manifests."
